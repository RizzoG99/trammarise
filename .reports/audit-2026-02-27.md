#### C-1: Undefined CSS Tokens in Shipped Components

**Category**: Theming
**Location**: 3 files
**WCAG**: N/A (design system integrity)

| Token Used                          | Used In                                           | Correct Token                |
| ----------------------------------- | ------------------------------------------------- | ---------------------------- |
| `var(--color-bg)`                   | `OnboardingPage.tsx:76`, `OnboardingPage.tsx:199` | `var(--color-bg-primary)`    |
| `var(--color-background-secondary)` | `SummaryPanel.tsx:145,156,175,184`                | `var(--color-bg-secondary)`  |
| `var(--color-text-muted)`           | `StepIndicator.tsx:52,64`                         | `var(--color-text-tertiary)` |

**Impact**: These tokens are not defined in `index.css`. Browsers silently fall back to `unset`/transparent ŌĆö no runtime error surfaces. If the design system ever removes these token names (currently undefined but not blocking), backgrounds vanish and text becomes invisible. Already causes invisible placeholder text in the API key input (`OnboardingPage`) and broken blockquote/table styling in `SummaryPanel`.

**Recommendation**: Replace all 3 stale token references with the correct canonical names from `index.css`.
**Suggested command**: `/normalize`

#### C-2: Two Competing Primary Colors

**Category**: Theming / Anti-Patterns
**Location**: 16+ files

The design system declares `--color-primary: oklch(54% 0.21 260)` (Ōēł `#256af4`). However, interactive elements across the codebase use `bg-indigo-600` (= `#4f46e5`) ŌĆö a visually distinct, darker purple-blue. Both appear simultaneously in the UI.

**Files with wrong primary** (sample):

- `src/lib/components/form/RadioCard/RadioCard.tsx:96,104` ŌĆö `border-indigo-600`, `bg-indigo-600`
- `src/lib/components/form/SelectCard/SelectCard.tsx:95,132` ŌĆö `bg-indigo-50 dark:bg-indigo-900/20 border-indigo-600`
- `src/lib/components/form/ToggleSwitch/ToggleSwitch.tsx:101` ŌĆö `bg-indigo-600 border-indigo-600`
- `src/lib/components/audio/PlaybackControls/PlaybackControls.tsx:33` ŌĆö `bg-indigo-600 hover:bg-indigo-700`
- `src/features/credits/CreditPurchaseModal.tsx:258` ŌĆö `border-indigo-600`

**Impact**: Every toggle, radio, select card, and audio playback button shows a different blue than the primary CTA button. The brand primary color is defined but effectively unused outside of recently refactored components.

**Recommendation**: Replace all `indigo-600` usages in interactive states with `[var(--color-primary)]` / `[var(--color-primary-hover)]`.
**Suggested command**: `/normalize`

---

#### C-3: Core Form Components Bypass Design System

**Category**: Theming
**Location**: `src/lib/components/form/` (RadioCard, SelectCard, ToggleSwitch, ConfigurationForm)

`RadioCard`, `SelectCard`, and `ToggleSwitch` are exported from `@/lib` but contain no CSS variable references at all ŌĆö they use raw Tailwind semantic colors (`slate-*`, `indigo-*`). These components are supposed to be the design system primitives.

**Impact**: Any theme change (e.g., switching `--color-primary`) has zero effect on these form controls. They are frozen to their initial Tailwind colors regardless of design system updates.

**Recommendation**: Refactor selected states to use `var(--color-primary)`, `var(--color-primary-alpha-10/20)`. Hover states use `var(--color-bg-secondary)`.
**Suggested command**: `/normalize`

## Patterns & Systemic Issues

### 1. Dual Color System Split

The design system has a clear canonical color in `--color-primary` (oklch `#256af4`), but **16+ files** use `indigo-600` (`#4f46e5`) via Tailwind's hardcoded palette. This split runs through form controls, audio controls, and marketing pages. **Root cause**: the CSS variable system was introduced during the current refactoring branch but the pre-existing components were not migrated.

### 2. Undefined Token Debt

3 token names are used in code but not defined in the CSS:

- `--color-bg` (bare, without suffix)
- `--color-background-secondary` (old naming convention)
- `--color-text-muted` (old naming convention)

These should be audited after each refactoring session using: `grep -r "var(--color-" src/ | grep -v "node_modules"` and cross-referenced against `index.css`.

### 3. Legacy Component Layer

`src/pages/`, `src/components/forms/`, `src/components/modals/`, and `src/components/states/` form a "legacy layer" that pre-dates the design system refactor. These components don't import from `@/lib` and contain the majority of hardcoded color and gradient issues. They need systematic migration, not piecemeal fixes.

### 4. Gradient Text Overuse

Gradient text appears in 5 separate locations (WelcomePage, PreviewPage, InitialState, OnboardingBanner, PricingPage). This is the single strongest "AI-generated" visual tell. It should be treated as a banned pattern going forward.

## Recommendations by Priority

### 1. Immediate (this sprint)

- **Fix undefined CSS tokens** (C-1): 3 files, ~10 lines total. Zero-risk fix. Prevents invisible UI elements.
- **Fix Button secondary/ghost/circle variants** (H-2): Align to CSS vars; fix `rounded-full` inconsistency.
- **Remove gradient text anti-patterns** (H-1): Replace with plain `text-[var(--color-text-primary)]` ŌĆö makes pages look immediately more trustworthy.

### 2. Short-term (next sprint)

- **Migrate core form lib components** (C-2, C-3): RadioCard, SelectCard, ToggleSwitch ŌĆö these are exported primitives and highest-impact for brand coherence.
- **Fix PricingCard and UpgradeModal** (H-3, H-4): Marketing-facing, high visibility.
- **Move ProcessAudio CTA** (H-6): UX improvement, single refactor in UploadRecordPage.
- **Close ARIA gaps** (M-1, M-2): CollapsibleSection aria-expanded, icon-only button aria-labels.

### 3. Medium-term (next 2 sprints)

- **Migrate legacy component layer** (M-5, H-7, H-9): ConfigurationForm, PlaybackControls, InitialState, ProcessingState.
- **Drop zone redesign** (H-7): Audio-specific visual identity.
- **Remove animated blob** (H-9): WelcomePage background.

### 4. Long-term (design system hardening)

- **Lint rule for undefined token detection**: Catch `--color-background-secondary`, `--color-bg` patterns at CI time.
- **Lint rule to ban `indigo-*` classes**: Enforce the single primary color at authoring time.
- **Refactor test assertions** (L-1): Replace CSS class assertions in SelectCard tests with semantic/behavioral assertions.
